# v0.5.0 - Vercel + QStash scheduler

## Summary

This release migrates the live Bus 15 Telegram alert off Supabase Cron and onto a Vercel + QStash + Upstash Redis setup.

## Added

- Vercel Functions implementation for:
  - `/api/health`
  - `/api/check`
  - `/api/telegram`
- QStash schedule setup script for the weekday morning checks.
- Stable QStash schedule IDs:
  - `bus15-alert-8am-sgt`
  - `bus15-alert-9am-sgt`
- Upstash Redis pause-state storage for same-day Telegram pause commands.
- Node-based test coverage for the Vercel implementation.

## Changed

- Vercel is now the live app/runtime.
- QStash is now the live scheduler.
- Supabase Cron is disabled and no longer sends scheduled alerts.
- README now documents the current production setup and live testing flow.

## Verified

- Live Telegram test sent successfully through `https://bus15-telegram-alert.vercel.app/api/check`.
- QStash schedules were created and verified active.
- Vercel dry-run returned `200 OK`.
- Vercel runtime errors check showed no recent errors.

## Notes

- Vercel Hobby cron cannot support this repeated weekday morning schedule, so QStash is used instead.
- Do not commit `.env`, Telegram bot tokens, LTA AccountKeys, Redis tokens, QStash tokens, or `CRON_SECRET` values.
