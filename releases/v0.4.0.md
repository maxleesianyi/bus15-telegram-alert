# v0.4.0 - Telegram Pause Commands

## Added

- Added Telegram reply commands to pause alerts for the current Singapore calendar day.
- Added `stop`, `pause`, and `done` commands.
- Added `resume` and `start` commands to reactivate alerts for the same day.
- Added Supabase table `bus15_daily_pauses` to store date-based pause state.
- Added Telegram webhook handling to the existing `bus15-telegram-alert` Edge Function.
- Added safety so only the configured `TELEGRAM_CHAT_ID` can control pause/resume behavior.

## Changed

- Supabase Cron runs now check whether today is paused before sending an alert.
- When paused, scheduled runs stay silent instead of sending "skipped" Telegram messages.

## Setup Notes

- Register the Telegram webhook once after deploying this version.
- Use the same `CRON_SECRET` as the Telegram webhook `secret_token`.
- Do not commit `.env`, Telegram bot tokens, LTA AccountKeys, or `CRON_SECRET` values.
